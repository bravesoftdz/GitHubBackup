JD GitHub Backup

Download Bulk Repositories from GitHub at Once

 

Summary

This application provides a very simple user interface which allows virtually anybody to be able to download many repositories at once. Currently, it is only to set up to download a ZIP file of the latest code in the default branch. Future versions will support multiple branches and more data downloaded in various formats.

 

Setup Your Account 

You must first configure the application before you can use it. This requires obtaining a personal access token from your GitHub account developer settings, and using that token here, if you want to access any private repositories. Otherwise, without this, you will have public visibility and limited access.  You must go to your account setup in this application, and enter that token. To open the setup, go to the "Options" menu at the top, and then choose "Setup".

 

 

Choose Source Account

You need to specify either a user or an organization by their unique name. Enter that into the repository setup. The application will display a list of all repositories in that user or organization which are accessible from the account behind the selected account .

 

Choose Download Location

You will also need to choose a folder somewhere on your computer or network where you wish to download these files. By default, this is in your "My Documents" folder, but you probably want to choose another location.

 

Listing Repositories

The main screen of the application allows you to fetch a complete list of all repositories you have access to under any given user or organization account. You can apply sorting, and later you will be able to filter them too.

 

Choosing Repositories

You can place a check next to any repository you wish to back up, or you can also check all. The selection will allow you to bulk-download ZIP files for each repository all at once.

 

Downloading Repositories

Once you have made your selection, you can click the "Download" button to initiate the download . This part will take time, depending on the number of repositories and their size. Note that the "size" GitHub reports is not accurate, and therefore there's no real chance of implementing estimated time.
